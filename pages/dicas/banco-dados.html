Banco de Dados

SQL, modelagem, otimização de consultas e estratégias de backup.

1. Normalize antes de codar

Aplique as 3 primeiras formas normais para evitar redundância. Uma tabela cliente separada de pedido evita dados duplicados.

Modelagem

2. Use índices em colunas de busca

Consultas com WHERE ou JOIN frequentes ganham velocidade com índices, mas não exagere: eles tornam os inserts mais lentos.

CREATE INDEX idx_pedido_cliente
ON pedido(cliente_id);
SQLPerformance

3. Evite SELECT * em produção

Traga só o que precisa. SELECT nome, email FROM usuario é mais leve e claro que SELECT *.

SQL

4. Backup automático e testado

Agende backups diários e, pelo menos uma vez por mês, restaure um backup em ambiente de teste para confirmar que funciona.

mysqldump -u root -p banco > backup_$(date +%F).sql
Backup

5. Use transações em operações críticas

Garanta consistência com BEGIN TRANSACTION / COMMIT / ROLLBACK em transferências e atualizações em lote.

SQLIntegridade

6. Conheça os tipos de JOIN e quando usar cada um

INNER JOIN retorna registros com correspondência em ambas as tabelas. LEFT JOIN retorna todos da esquerda, mesmo sem correspondência. RIGHT JOIN e FULL JOIN completam o quadro. Escolha errada custa performance.

SQLConsultas

7. Use EXPLAIN para analisar consultas lentas

O comando EXPLAIN mostra como o banco executa sua consulta: quais índices usa, quantas linhas examina, etc. Com isso você identifica gargalos e otimiza.

EXPLAIN SELECT * FROM pedidos
WHERE data BETWEEN '2025-01-01' AND '2025-12-31';
OtimizaçãoSQL

8. Modele pensando no futuro: evite campos booleanos

Em vez de ativo BOOLEAN, prefira status VARCHAR(20) com valores como 'ativo', 'inativo', 'suspenso'. Isso permite evoluir sem alterar a estrutura da tabela.
